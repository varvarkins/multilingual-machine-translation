# B. Русско-Абхазский переводчик

Абхазский язык — один из самых интересных и сложных языков Кавказа. Это малоресурсный язык: для него мало параллельных корпусов, ограниченная языковая поддержка в популярных сервисах перевода и немного готовых решений в области машинного обучения. Помимо этого, большинство языковых моделей даже не знают его уникальных букв алфавита - ӷӡқҟԥҭҳҵҷҽҿҩџьәҕҧ.

Главная задача: создать модель машинного перевода с русского на абхазский, которая покажет максимально возможный результат по автометрие BLEU (bilingual evaluation understudy) на нашем тестовом наборе.

В данной задаче можно использовать любые архитектуры моделей, заниматься дополнительным майнингом данных, создавать синтетические корпуса и экспериментировать с языковыми группами - всё может помочь!

Мы уже подготовили для вас параллельные и моноязычные корпуса чтобы вам было проще начать обучение:

- Параллельный корпус (русский <-> абхазский): [ссылка на Яндекс Диск](https://disk.yandex.ru/d/2NqWWLCwTpFl1w)
- Моноязычные данные (абхазский): [ссылка на Яндекс Диск](https://disk.yandex.ru/d/kA5prRXUDpf6fg)

## Формат ввода

Ваше решение — это docker-контейнер. Мы запускаем его, подкладываем файл с вопросами, ждём файл с ответами.

Вам нужно прислать zip-архив с решением в задачу.

### Содержимое zip-архива

Минимально:

```
Dockerfile        # описывает окружение
solution.py       # точка входа — запускается при старте контейнера
weights/          # веса вашей модели
├── config.json
├── tokenizer.json
├── model.safetensors
└── ...
```

Можно добавлять любые свои файлы — скрипты, дополнительные данные, ещё одну модель. Всё, что попадёт в zip, окажется в папке /workspace внутри контейнера.

Важно — как правильно собрать архив:

- Лучше всего: «плоский» архив — Dockerfile, solution.py и weights/ лежат прямо в корне zip-файла, без обёрточной папки. Это именно то, как разложен скачанный вами шаблон — просто выделите все файлы внутри папки и упакуйте их.
- Тоже примем: если у вас весь набор обёрнут в одну папку (например, my-solution/Dockerfile, my-solution/solution.py, …) — система автоматически «поднимет» содержимое этой папки наверх. Так часто получается, когда на macOS делаете «правый клик → Сжать»: macOS оборачивает выбранное в папку с именем zip-файла.
- Не примем: двойную обёртку (my-solution/my-solution/Dockerfile) — здесь автомат уже не угадает, что чей-то файл, и вы получите PresentationError. Если такое произошло — переупакуйте содержимое папки, а не саму папку.

### Файл с вопросами — input.pickle

Файл формата input.pickle, где src это текст на русском, а rid это айди который надо передать далее. Тестовый набор содержит 515 примеров, средняя длина 78 символов (12 слов), в каждом примере не больше трёх предложений.

Формат файла:

```
[
    {
        "rid": int,
        "src": str,
    },
]
```

## Формат вывода

Ваш код должен положить в output.json JSON-список такого вида:

- где translation это перевод на абхазском
- где rid это айди строчки из input.pickle файла


## Система оценивания

Балл за попытку - среднее арифметическое по автометрике BLEU (bilingual evaluation understudy) на закрытом тестовом корпусе. Метрика считается на каждом примере через библиотеку sacrebleu, финальная оценка это среднее арифметическое по всем переводам. Старайтесь чтобы модель не генерировала дополнительные комментарии "Вот перевод с русского на абхазский", так как они негативно скажутся на результат.

Вам даётся по 4 попытки каждые 24 часа.

## Примечание

### С чего начать

1. Скачайте этот архив — в нём уже лежит рабочий пример решения (solution.py, Dockerfile, download_weights.py, example-input.pickle, example-output.json).
2. Установите Docker.
3. Скачайте веса модели-бейзлайна в папку ./weights (см. «Подготовка весов»).
4. Соберите и прогоните на примере (важно: уберите флаг --gpus all если нет видеокарты на машинке):

```bash
docker build -t my-solution .
mkdir -p out
docker run --rm --gpus all \
  -v "$PWD/example-input.pickle":/workspace/input.pickle \
  -v "$PWD/out":/workspace/out \
  my-solution
cat out/output.json
```

5. Упакуйте содержимое папки в zip (включая weights/) и загрузите на contest.yandex.ru.

В Dockerfile уже лежит образ из внутреннего реестра Яндекса (container-registry). Можно заменить образ на любой другой, но настоятельно рекомендуем использовать его (он собирается быстрее, для него не нужна сторонняя авторизация, мы даём гарантию что он рабочий).

### Бейзлайн: gemma-4-E2B

В архиве уже есть рабочее решение. Оно поднимает поднимает gemma4-E2B-it модель, прогоняет через неё все вопросы из input.pickle и пишет ответы в output.json. Ни специального промпта, ни дополнительных трюков в нём нет — это нижняя планка, с которой удобно сравнивать улучшения.

Этого достаточно, чтобы отправить первое решение и увидеть свой балл. Учтите - Gemma-4 не знает абхазский язык. Мы советуем поизучать какие LLM/переводные модели поддерживают его, или попробовать обучить свою с нуля.

Также подскажем, что решение которое просто копирует русские тексты оценивается в 5.2 BLEU. А лучший на сейчас нам подход оценивается в 17.6 BLEU, сможете побить?

### Подготовка весов

Запустите один раз локально:

```bash
python download_weights.py
```

Скрипт положит веса Gemma4 (~10 ГБ) в папку ./weights. Её нужно включить в ваш zip-архив — без весов контейнер внутри контеста ничего не сможет скачать (сети нет).

### Ограничения

- Контейнер запускается без доступа к интернету. Всё нужное (веса моделей, пакеты, данные) должно лежать в образе или архиве — скачать на лету ничего не получится.
- Время работы скрипта генерации ответов на 515 запросов должно занимать не более 30 минут.
- Ваш образ должен занимать менее 20GB.
- Решение будет запускаться на сервере с одной картой L4 (24 GB VRAM), 16GB RAM, 93 GB SSD IO, 4 vCPU, платформа Intel Ice Lake.
